# Hiring Philosophy for Repo-Understanding Interviews in the Age of Agents

## Why this interview exists

The scarce skill is no longer raw code production. It is the ability to understand an unfamiliar system quickly, choose a sound approach, use agents without giving up ownership, and leave behind code that can be tested, benchmarked, and improved over time.

## What strong candidates do

Strong candidates usually:

- form a system model before delegating
- use agents to accelerate understanding, not replace judgment
- compare multiple algorithm or implementation strategies before committing
- make one coherent change instead of many scattered patches
- build confidence with tests, measurements, and targeted evidence
- leave behind tools they or a teammate could use to self-heal and optimize later

## What weak candidates do

Weak candidates often:

- start coding before they understand the repo
- ask AI to solve the whole problem before tracing the code path
- choose the first plausible algorithm without comparison
- ship something that works once but cannot be trusted or tuned
- treat benchmarking or tests as optional polish instead of future leverage

## What this interview is screening for

This loop is designed to measure six traits:

1. `Repo understanding`
2. `Agent orchestration`
3. `Algorithm judgment`
4. `Safe implementation`
5. `Evidence and self-healing mindset`
6. `Ownership and communication`

## Decision rule

Hire people who can think before they delegate, use multiple agents with clear boundaries, choose sensible algorithms, and create lasting confidence through tests and benchmarks.
